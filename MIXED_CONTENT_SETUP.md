# Настройка браузера для разрешения Mixed Content

Если приложение развернуто на HTTPS (например, GitHub Pages), но API работает на HTTP, браузер будет блокировать запросы из-за политики Mixed Content.

## Решение: Настройка браузера для разрешения Mixed Content

### Chrome/Edge:

1. Откройте сайт с приложением
2. Нажмите на иконку замка в адресной строке
3. Выберите "Настройки сайта"
4. В разделе "Небезопасный контент" выберите "Разрешить"
5. Перезагрузите страницу

Или через флаги Chrome:
1. Перейдите в `chrome://flags/#unsafely-treat-insecure-origin-as-secure`
2. Добавьте URL вашего сайта (например, `https://yourusername.github.io/smartmoney-client`)
3. Перезапустите браузер

### Firefox:

1. Откройте сайт с приложением
2. Нажмите на иконку замка в адресной строке
3. Нажмите "Подробнее"
4. Нажмите "Отключить защиту на этой странице"
5. Перезагрузите страницу

Или через настройки:
1. Введите в адресной строке `about:config`
2. Найдите `security.mixed_content.block_active_content`
3. Установите значение `false`
4. Перезагрузите страницу

### Альтернативное решение:

Настроить сервер для работы через HTTPS или использовать прокси-сервер, который будет проксировать HTTP-запросы через HTTPS.
